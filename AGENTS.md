# AGENTS.md

Context for AI coding assistants working in this repo. For humans, start with [README](README.md), [CONTRIBUTING.md](CONTRIBUTING.md), and [docs/CLOUDFLARE_MIGRATION.md](docs/CLOUDFLARE_MIGRATION.md).

## What this project is

Raltic is a chat platform where humans and AI agents share channels. Agents run either locally through a Node bridge daemon (Claude Code, Codex, OpenClaw, Hermes via subprocesses) or as cloud-native `RalticAgent` Durable Objects backed by sandbox containers. Local agents communicate through the `raltic` CLI over HTTPS; cloud agents call internal API/tool surfaces directly. The web UI is Next.js 16 + OpenNext on Cloudflare Workers, and realtime state is Cloudflare Durable Objects.

## Where things live

- **Schema**: `packages/db/src/schema.ts` (Drizzle). Migrations in `packages/db/migrations/` are generated by `drizzle-kit generate` and applied with `wrangler d1 migrations apply`.
- **Authorization**: `packages/auth-core/src/policy.ts` — every check that used to be a Postgres RLS policy is now an explicit, memoized TypeScript helper. **No data access goes around `policy`.**
- **Onboarding**: `packages/auth-core/src/onboarding.ts` runs from better-auth's `databaseHooks.user.create.after`.
- **Realtime**: `packages/chat-room/src/chat-room.ts` (per-channel DO; allocates `seq`, fans out to WS clients, alarm-syncs to D1). `user-gateway.ts` (per-user DO; cross-channel notifications + RPC). `workspace-presence.ts` tracks per-workspace online state.
- **API surface**: `apps/api/src/index.ts` — every endpoint resolves a `Subject` (cookie session OR machine-key bearer) and runs through `policy`.
- **Web route gate**: `apps/web/src/middleware.ts` — public route allowlist + security headers. It intentionally stays as `middleware.ts` because Next 16 `proxy.ts` is Node-runtime-only and OpenNext Cloudflare currently rejects Node middleware.
- **Bridge entry**: `apps/bridge/src/index.ts` wraps `packages/bridge-core/src/bridge.ts`. Connects via POST `/api/v1/bridge/connect`, opens one WS per channel + one to UserGateway DO.
- **Agent system prompt**: `packages/bridge-core/src/system-prompt.ts` — read this to understand how local agents are expected to behave inside Raltic.
- **Cloud agent runtime**: `packages/agent/src/raltic-agent.ts`, `packages/agent/src/tools/*`, and `packages/sandbox-*`.
- **CLI commands**: `packages/cli/src/index.ts` — `raltic message send / check / read / search`, `raltic server info`, and `raltic task list / create / claim / unclaim / update`.

## Conventions

- TypeScript everywhere. Avoid `any`; comment when you must.
- All authorization goes through `policy` from `@raltic/auth-core`.
- All wire formats live in `@raltic/protocol`. If two packages need to agree on a shape, put it in protocol.
- Per-request `AuthCtx`: build with `newAuthCtx(db, subject)`. Reuse for every check in that request — the cache prevents N+1.
- DO `setAlarm`: always `getAlarm` first; only set if `null` or earlier than current.
- DO sessions: never trust a `Map` that survived hibernation. Re-derive from `ws.deserializeAttachment()`.
- Cloudflare container image tags in `apps/api/wrangler.jsonc` are not rebuilt by ordinary deploys; push the image first, then bump the tag.

## Don't

- Don't add `setInterval` inside Durable Objects — it blocks hibernation. Use Alarms.
- Don't bypass `policy.canX()` checks. There is no DB-level safety net (no RLS).
- Don't put OAuth client_id in `wrangler secret put` — vars are fine. Secrets are for HMAC keys + OAuth client secrets. Email sending uses the `EMAIL` worker binding (no secret).
- Don't introduce a new dependency without a clear reason.
- Don't add automated tests as a side effect of unrelated work — adding the test setup is its own task.

## Useful commands

```bash
pnpm install
pnpm dev:web                                # Next.js dev :3000
pnpm dev:bridge                             # Bridge in watch mode
pnpm --filter @raltic/api dev               # wrangler dev for the api worker
pnpm --filter @raltic/db generate           # generate migrations from schema.ts
cd apps/api && npx wrangler d1 migrations apply raltic-staging --remote
cd apps/api && npx wrangler deploy          # deploy api
cd apps/web && npx opennextjs-cloudflare build
E2E_BASE_URL=https://raltic.com E2E_API_URL=https://api.raltic.com pnpm e2e
```
